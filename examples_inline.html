Various Examples for jsonToD3.js (Inline JSON)

In the examples here, data is described in-line in contrast to the examples in examples_load_by_src.html where they are loaded from an external file specified by the SRC attribute. Check those examples out. They will be of interest to most, especially Singaporeans.

For those who are interested in jsonToD3.js as a charting library, check out the GitHub repository for jsonToD3.js.

The Chart

Points in a data series are usually in the form [x, y] (or [x, y, z]), but may be given names as in [x, y, name] (or [x, y, z, name]). (Have a look at Series $\beta$ starting from the second point.)

In fact, one can go a step further and determine the value displayed in the tooltip, usually (x, y) and with entries like as [x, y, name, display_value] (or [x, y, z, name, display_value]), replace the default with display_value. One thing to note is that the point has to be named. (Empty string names will be treated as non-names.) Try to use a string for it outright. (See this at work in the points on the left.)

MathJax has been used for typesetting math. (The solution is rather hackish for now. But unfortunately that might not change.) Some equations take a bit longer to typeset, unfortunately, try doing a mouseover on the second big blue marker.

But it is ameliorated by enabling the user to add an additional entry in the data for it's "display value", as mentioned in a previous example. Again, it might be useful to make that a string directly. (See all the (H) series. Also note the discrepancy due to rounding.)
